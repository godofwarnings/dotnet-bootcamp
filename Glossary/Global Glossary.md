# Global Glossary

A centralized glossary of terminology used throughout the bootcamp.

## A
- **[Agile](../Concepts/Agile/Agile%20Basics.md):** An iterative approach to software development emphasizing flexible planning and early delivery.
- **API (Application Programming Interface):** The business logic layer that acts as an intermediary between the UI and DB.
- **[Async](../Concepts/Async%20Programming/Async%20vs%20Sync.md):** A programming model that allows non-blocking operations.

## C
- **[Class](../Concepts/C%23/Classes%20and%20Objects.md):** A blueprint for creating objects, containing state and behavior.
- **[ConcurrentBag](../Concepts/Async%20Programming/Thread%20Safe%20Collections.md):** A thread-safe, unordered collection of objects optimized for scenarios where the same thread is both producing and consuming data.
- **[ConcurrentQueue](../Concepts/Async%20Programming/Thread%20Safe%20Collections.md):** A thread-safe, First-In-First-Out collection provided by .NET.

## D
- **[Dapper](../Concepts/Database/Dapper%20Basics.md):** A high-performance Micro-ORM for .NET that maps SQL results to C# objects.
- **[Delegate](../Concepts/C%23/Delegates%20and%20Action.md):** A type that safely encapsulates a method.
- **DRY (Don't Repeat Yourself):** A software principle aimed at reducing repetition of software patterns.

## E
- **[Entity Framework Core](../Concepts/Database/Entity%20Framework%20Core.md):** A full Object-Relational Mapper (ORM) that abstracts SQL behind C# code.
- **[Epics](../Concepts/Agile/Agile%20Basics.md):** Large bodies of work in Agile that can be broken down into user stories.
- **[Exception](../Concepts/C%23/Exception%20Handling.md):** An error that occurs during the execution of a program.

## I
- **[Interface](../Concepts/C%23/Interfaces%20and%20Abstract%20Classes.md):** A contract that defines a set of methods and properties a class must implement.

## L
- **[LINQ](../Concepts/C%23/LINQ%20Basics.md) (Language Integrated Query):** A unified syntax to query and manipulate data directly from C#.
- **[lock](../Concepts/Async%20Programming/Parallel%20Programming.md):** A statement used to ensure that only one thread can execute a critical section of code at a time.

## O
- **ORM (Object Relational Mapper):** A tool that automates mapping database tables to C# objects.

## P
- **[Parallel.For / Parallel.ForEach](../Concepts/Async%20Programming/Parallel%20Programming.md):** Methods to execute loops in parallel using multiple threads.
- **[Publisher-Subscriber](../Concepts/Architecture/Design%20Patterns.md):** A messaging pattern where senders don't address messages directly to specific receivers.

## R
- **[Repository Pattern](../Concepts/Architecture/Design%20Patterns.md):** A design pattern that acts as a middleman between the application's business logic and data access layers.

## S
- **[Singleton](../Concepts/Architecture/Design%20Patterns.md):** A design pattern that ensures a class has only one instance and provides a global point of access to it.
- **[Static](../Concepts/C%23/Static%20Methods.md):** A modifier indicating that a member belongs to the class itself rather than a specific instance.
- **SqlDataReader:** An ADO.NET class used to read a forward-only stream of rows from a database.

## T
- **[Task](../Concepts/Async%20Programming/Task%20vs%20Thread.md):** A promise of future completion or representation of an operation (not necessarily a thread).
- **Task.WhenAll:** An async method that acts as a coordinator, waiting for multiple tasks to finish.
- **[Thread](../Concepts/Async%20Programming/Task%20vs%20Thread.md):** A basic unit of CPU utilization.

## U
- **[User Story](../Concepts/Agile/Agile%20Basics.md):** A short, simple description of a feature from the user's perspective.
