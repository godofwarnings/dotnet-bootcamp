### Repository pattern

- repository pattern

#### [image desccirption]

This image illustrates the Repository Pattern in software architecture, specifically contrasting a disorganized "direct access" approach with a structured "Repository Layer" approach.

Diagram Components

1. Client Side (Left)
   - Classes: There are three rectangular boxes representing application classes.
   - Scale: A text label indicates "500 classes," suggesting a large-scale system where managing data access across many entities is complex.

2. Repository Layer (Center)
   - Abstraction: A large central container labeled "Repository Layer" acts as a mediator between the application classes and the data sources.
   - Operations: It explicitly lists standard CRUD operations: Insert, Update, Select, and Delete.
   - Flow: Block arrows indicate that the 500 classes should communicate with this layer rather than the databases directly.

3. Data Sources (Right)
   - Relational DB: An oval labeled "DB" containing MSSQL.
   - NoSQL DB: An oval containing Mongo DB.

Key Visual Concept: The "Wrong" vs. "Right" Way

- Direct Access (The Problem): Hand-drawn squiggly lines show the classes trying to connect directly to the MSSQL database, bypassing the Repository Layer.
- The "X" Mark: A large "X" is drawn over these direct connections. This signifies that allowing 500 different classes to write their own SQL queries directly to the database is a "bad practice" or a "bottleneck" that leads to messy, unmaintainable code.
- The Solution: The diagram suggests that by using the Repository Layer, the application can decouple the business logic from the specific database implementation (MSSQL or Mongo DB), making the system easier to maintain and switch between different data providers.

// StudentRepository.cs

using ConsoleApp4;
using Dapper;
using Microsoft.Data.SqlClient;
using System.Collections.Generic;
using System.Linq;

public class StudentRepository
{
    private readonly string connectionString =
        "Server=(localdb)\\MSSQLLocalDB;Database=EducationDB;Trusted_Connection=True;TrustServerCertificate=True;";

    public List GetAllStudents()
    {
        using SqlConnection connection = new SqlConnection(connectionString);

        string query = "SELECT Id, FirstName, LastName, Age, Email, Course FROM Student";

        List students = connection.Query(query).ToList();

        return students;
    }

    public Student? GetStudentById(int id)
    {
        using SqlConnection connection = new SqlConnection(connectionString);

        string query = "SELECT Id, FirstName, LastName, Age, Email, Course FROM Student WHERE Id = @Id";

        Student? student = connection.QueryFirstOrDefault(query, new { Id = id });

        return student;
    }

    public void InsertStudent(Student student)
    {
        using SqlConnection connection = new SqlConnection(connectionString);

        string query = @"INSERT INTO Student (FirstName, LastName, Age, Email, Course)
                         VALUES (@FirstName, @LastName, @Age, @Email, @Course)";

        connection.Execute(query, student);   
    }

    public void UpdateStudentEmail(int id, string email)
    {
        using SqlConnection connection = new SqlConnection(connectionString);

        string query = "UPDATE Student SET Email = @Email WHERE Id = @Id";

        connection.Execute(query, new {Email = email, Id = id});
    }

    public void DeleteStudent(int id)
    {
        using SqlConnection connection = new SqlConnection(connectionString);

        string query = "DELETE FROM Student WHERE Id = @Id";

        connection.Execute(query, new { Id = id });
    }
}
