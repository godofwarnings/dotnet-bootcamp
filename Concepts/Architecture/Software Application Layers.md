# Software Application Layers

A software application typically consists of **at least 3 layers**, separating concerns for better maintainability and scalability.

## 1. UI Layer (Presentation)
- Contains **presentation logic**.
- Responsible for what the user sees and interacts with.
- **Examples:**
  - HTML / CSS / JavaScript
  - Angular (Focus of the bootcamp)
  - ReactJS
  - EmberJS

## 2. API Layer (Business Logic)
- Contains the core **business logic** and rules of the application.
- Acts as the intermediary between the UI and the Database.
- **Examples:**
  - Web API (.NET) (Focus of the bootcamp)
  - ExpressJS (Node.js)
  - Spring (Java)
  - FastAPI / Django (Python)

## 3. Database Layer (DB)
- Contains the **data** and is responsible for data persistence.
- **Examples:**
  - Microsoft SQL Server (Focus of the bootcamp)
  - MySQL
  - Oracle
  - MongoDB
  - PostgreSQL
